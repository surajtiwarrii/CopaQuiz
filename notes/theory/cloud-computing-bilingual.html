Cloud Computing का परिचयIntroduction to Cloud Computing

Cloud Computing क्या है?

What is Cloud Computing?

Cloud Computing Internet के through computing services (storage, servers, software, processing power) को on-demand access करने का तरीका है — बिना खुद hardware खरीदे।

Cloud Computing is the delivery of computing services (storage, servers, software, processing power) over the Internet on-demand — without owning physical hardware.

Cloud Computing से पहले vs अब

Before Cloud Computing vs Now

पहले (On-Premise)Before (On-Premise)अब (Cloud)Now (Cloud)
अपना hardware खरीदना और maintain करना पड़ता थाHad to buy and maintain own hardwareसिर्फ रेंट पर ज़रूरत भर resources लेनाRent only the resources you need
High upfront costHigh upfront costPay-as-you-go pricingPay-as-you-go pricing

Cloud Computing की Key Characteristics

Key Characteristics of Cloud Computing

CharacteristicमतलबMeaning
On-Demand Self-ServiceUser बिना किसी provider से बात किए खुद resources ले सकता हैUsers can provision resources themselves without contacting the provider
Broad Network AccessInternet से किसी भी device से accessAccessible from any device over the Internet
Resource Poolingकई users के बीच resources share होते हैंResources are shared among multiple users
Rapid Elasticityज़रूरत के हिसाब से resources तुरंत बढ़ाए/घटाए जा सकते हैंResources can be quickly scaled up/down as needed

Three Main Service Models

ModelमतलबMeaningउदाहरणExample
IaaS
(Infrastructure as a Service)(Infrastructure as a Service)		Virtual servers, storage, networking rent पर मिलता हैProvides virtual servers, storage, networking on rentAWS EC2, Azure VM
PaaS
(Platform as a Service)(Platform as a Service)		App बनाने/run करने का पूरा platform मिलता है (infrastructure manage नहीं करना पड़ता)Provides a complete platform to build/run apps (no need to manage infrastructure)Google App Engine, Heroku

Responsibility Comparison

Responsibility Comparison

IaaS में user को सबसे ज़्यादा control और responsibility (OS, apps खुद manage करनी पड़ती है) मिलती है। SaaS में सबसे कम — user सिर्फ software use करता है, बाकी सब provider manage करता है।

IaaS gives the user the most control and responsibility (managing OS, apps themselves). SaaS gives the least — the user only uses the software while the provider manages everything else.

The "As a Service" Pizza Analogy

ModelPizza AnalogyPizza Analogy
On-Premiseघर पर pizza बनाना (सब कुछ खुद)Making pizza at home (everything yourself)
IaaSKitchen rent पर लेना (ingredients खुद लाने पड़ते हैं)Renting a kitchen (still bring your own ingredients)
PaaSPizza takeaway-and-bake (आधा बना मिलता है)Take-and-bake pizza (half-prepared for you)
SaaSRestaurant में pizza order करना (पूरी तरह ready-made)Ordering pizza at a restaurant (fully ready-made)
याद रखें: IaaS < PaaS < SaaS के क्रम में user की responsibility कम होती जाती है और provider की responsibility बढ़ती जाती है। Remember: going from IaaS → PaaS → SaaS, the user's responsibility decreases while the provider's responsibility increases.

Hybrid Cloud का उपयोग

Use of Hybrid Cloud

Companies sensitive data को Private Cloud में रखती हैं और बाकी workload को Public Cloud पर — यह flexibility और security दोनों का बेहतरीन balance देता है।

Companies keep sensitive data in a Private Cloud and run the rest of the workload on a Public Cloud — giving the best balance of flexibility and security.

याद रखें: Public Cloud सबसे सस्ता और flexible है, पर Private Cloud सबसे secure है — Hybrid Cloud दोनों के फायदे combine करता है। Remember: Public Cloud is the cheapest and most flexible, but Private Cloud is the most secure — Hybrid Cloud combines benefits of both.

Virtualization क्या है?

What is Virtualization?

Virtualization एक physical computer पर कई virtual machines (VMs) बनाने की technology है — Cloud Computing इसी पर based है, जिससे एक hardware के resources कई users में share होते हैं।

Virtualization is the technology of creating multiple virtual machines (VMs) on one physical computer — Cloud Computing is built on this, allowing one hardware's resources to be shared among many users.

Virtual Machine vs Physical Machine

Virtual Machine vs Physical Machine

Virtual MachinePhysical Machine
Software-based, एक physical machine पर कई VMs चल सकती हैंSoftware-based, multiple VMs can run on one physical machineActual hardware, एक OS एक समय परActual hardware, one OS at a time

Hypervisor

Hypervisor

Hypervisor वो software layer है जो physical hardware पर virtual machines को create और manage करता है — VMware, VirtualBox, Hyper-V इसके उदाहरण हैं।

A Hypervisor is the software layer that creates and manages virtual machines on physical hardware — VMware, VirtualBox, Hyper-V are examples.

याद रखें: Virtualization के बिना modern Cloud Computing possible नहीं होता — यही technology resource pooling और elasticity को मुमकिन बनाती है। Remember: modern Cloud Computing wouldn't be possible without Virtualization — this technology enables resource pooling and elasticity.

Cloud Storage & DatabaseCloud Storage & Database

Cloud Storage क्या है?

What is Cloud Storage?

Cloud Storage data को Internet के through remote servers पर store करने की service है, जिसे कहीं से भी access किया जा सकता है — जैसे Google Drive, Dropbox, OneDrive।

Cloud Storage is a service that stores data on remote servers over the Internet, accessible from anywhere — like Google Drive, Dropbox, OneDrive.

Cloud Storage के फायदे

Advantages of Cloud Storage

  • कहीं से भी, किसी भी device से access
  • Automatic backup, data loss का कम risk
  • Easy file sharing और collaboration
  • Scalable — ज़रूरत के हिसाब से storage बढ़ा सकते हैं
  • Access from anywhere, any device
  • Automatic backup, lower risk of data loss
  • Easy file sharing and collaboration
  • Scalable — increase storage as needed

Cloud Database

Cloud Database

Cloud Database एक database है जो cloud platform पर run होता है, बिना खुद server manage किए — जैसे Amazon RDS, Google Cloud SQL, Firebase।

A Cloud Database is a database that runs on a cloud platform without needing to manage the server yourself — like Amazon RDS, Google Cloud SQL, Firebase.

DBaaS (Database as a Service)

DBaaS (Database as a Service)

DBaaS Cloud Provider द्वारा managed database service है — provisioning, scaling, backup सब automatically handle होता है।

DBaaS is a database service managed by the Cloud Provider — provisioning, scaling, backup are all handled automatically.

याद रखें: Cloud Storage files के लिए होता है (Google Drive), जबकि Cloud Database structured data के लिए (rows/columns) — दोनों अलग use cases हैं। Remember: Cloud Storage is for files (Google Drive), while a Cloud Database is for structured data (rows/columns) — both serve different use cases.

Cloud SecurityCloud Security

Cloud Security क्यों ज़रूरी है?

Why is Cloud Security Important?

Cloud पर data third-party servers पर store होता है, इसलिए unauthorized access, data breach, और loss से बचाने के लिए strong security measures ज़रूरी हैं।

Since data is stored on third-party servers in the cloud, strong security measures are essential to prevent unauthorized access, data breaches, and loss.

Shared Responsibility Model

Best Practices

Best Practices

  • Strong authentication (2FA) use करें
  • Data को encrypt करके store करें
  • Access permissions को regularly review करें
  • Regular backups लें
  • Use strong authentication (2FA)
  • Store data encrypted
  • Regularly review access permissions
  • Take regular backups
Exam Tip: Shared Responsibility Model exam में अक्सर पूछा जाता है — याद रखें Provider infrastructure secure करता है, Customer data/access secure करता है। Exam Tip: the Shared Responsibility Model is often asked in exams — remember the Provider secures infrastructure, the Customer secures data/access.

SDLC (Software Development Life Cycle)SDLC (Software Development Life Cycle)

SDLC क्या है?

What is SDLC?

SDLC software/app बनाने का एक structured process है, जो planning से लेकर maintenance तक के सभी phases को cover करता है।

SDLC is a structured process for building software/apps, covering all phases from planning to maintenance.

SDLC के Phases

Phases of SDLC

PhaseकामDescription
Requirement AnalysisApp में क्या चाहिए, यह समझनाUnderstanding what's needed in the app
DesignApp की architecture और UI/UX plan करनाPlanning the app's architecture and UI/UX
Development (Coding)Actual code लिखनाWriting the actual code
TestingBugs/errors ढूंढना और fix करनाFinding and fixing bugs/errors
DeploymentApp को users के लिए live/launch करनाLaunching the app live for users
MaintenanceLaunch के बाद updates और bug fixes देनाProviding updates and bug fixes after launch

SDLC Models

SDLC Models

ModelविशेषताFeature
Waterfall Modelहर phase sequentially पूरा होता है, पीछे जाना मुश्किलEach phase completes sequentially, hard to go back
Agile Modelछोटे-छोटे iterations (Sprints) में काम होता है, flexibleWork happens in small iterations (Sprints), flexible
याद रखें: Waterfall एक linear process है (एक phase के बाद ही दूसरा शुरू होता है), जबकि Agile में continuous feedback और changes accommodate किए जाते हैं। Remember: Waterfall is a linear process (one phase starts only after another ends), while Agile accommodates continuous feedback and changes.

Challenges of Cloud Computing

  • Data security और privacy की चिंता
  • Provider पर dependency (Vendor Lock-in)
  • Downtime का risk

  • Dependency on the provider (Vendor Lock-in)
