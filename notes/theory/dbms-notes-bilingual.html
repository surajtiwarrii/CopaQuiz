Unit 01 · शुरुआतUnit 01 · Basics

DBMS परिचय (Introduction)DBMS Introduction

Data, Information और Database क्या है?

What is Data, Information and Database?

Data कच्चे facts और figures होते हैं जिनका अभी कोई मतलब नहीं निकला होता — जैसे "25", "Suraj", "Gonda"। जब इस data को व्यवस्थित करके किसी रूप में लाया जाता है तो वो Information बन जाता है — जैसे "Suraj की उम्र 25 साल है और वो Gonda में रहता है।"

Data consists of raw facts and figures with no meaning extracted yet — like "25", "Suraj", "Gonda". When this data is organized into some form, it becomes Information — like "Suraj is 25 years old and lives in Gonda."

Database related data का एक संगठित collection होता है जिसे आसानी से access, manage और update किया जा सकता है।

A Database is an organized collection of related data that can be easily accessed, managed, and updated.

DBMS की परिभाषा

Definition of DBMS

DBMS (Database Management System) एक software होता है जो database को बनाने, उसमें data store करने, उसे manage करने और retrieve करने का काम करता है। यह user और database के बीच एक interface का काम करता है।

A DBMS (Database Management System) is software used to create databases, store data, manage it, and retrieve it. It acts as an interface between the user and the database.

याद रखें: DBMS = Software जो Database को Manage करता है। Examples: MySQL, Oracle, MS Access, SQL Server, MongoDB Remember: DBMS = Software that manages a Database. Examples: MySQL, Oracle, MS Access, SQL Server, MongoDB

DBMS के लाभ (Advantages)

Advantages of DBMS

  • Data Redundancy कम होती है — एक ही data बार-बार store नहीं होता।
  • Data Sharing आसान — कई users एक साथ data इस्तेमाल कर सकते हैं।
  • Data Security — passwords और permissions से data सुरक्षित रहता है।
  • Data Integrity — data सही और सटीक बना रहता है।
  • Backup और Recovery — system crash होने पर data वापस मिल सकता है।
  • Reduced Data Redundancy — the same data is not stored repeatedly.
  • Easy Data Sharing — multiple users can use the data at the same time.
  • Data Security — data stays protected via passwords and permissions.
  • Data Integrity — data remains accurate and correct.
  • Backup and Recovery — data can be recovered after a system crash.

DBMS के नुकसान (Disadvantages)

Disadvantages of DBMS

  • Software और hardware की cost ज़्यादा होती है।
  • System को चलाने के लिए trained staff चाहिए होता है।
  • Complex system होने की वजह से सीखना मुश्किल होता है।
  • Software and hardware costs are higher.
  • Trained staff is needed to run the system.
  • It is difficult to learn due to the system's complexity.

  • Hardware — physical devices like computer, storage

Unit 02 · शुरुआतUnit 02 · Basics

Data Models (डेटा मॉडल)Data Models

Data Model क्या है?

What is a Data Model?

Data Model एक concept या तरीका है जिससे यह बताया जाता है कि data को कैसे store, organize और access किया जाएगा। यह database की संरचना को logically represent करता है।

A Data Model is a concept or method that describes how data will be stored, organized, and accessed. It logically represents the structure of the database.

Data Models के प्रकार

Types of Data Models

  • Hierarchical Model — data को tree जैसी structure में रखा जाता है, parent-child relationship होती है। एक parent के कई child हो सकते हैं पर एक child का सिर्फ एक parent होता है।
  • Network Model — Hierarchical model का improved version, इसमें एक child के कई parents हो सकते हैं (graph structure)।
  • Relational Model — सबसे ज़्यादा इस्तेमाल होने वाला model, data को rows और columns वाली tables में रखा जाता है। E.F. Codd ने इसे 1970 में प्रस्तावित किया था।
  • Object-Oriented Model — data को objects के रूप में रखा जाता है, जैसे OOP programming में होता है।
  • E-R Model — database design के लिए इस्तेमाल होता है, entities और उनके बीच relationships को दिखाता है।
  • Hierarchical Model — data is kept in a tree-like structure, with parent-child relationships. A parent can have multiple children, but a child has only one parent.
  • Network Model — an improved version of the Hierarchical model, where a child can have multiple parents (graph structure).
  • Relational Model — the most widely used model, where data is kept in tables with rows and columns. Proposed by E.F. Codd in 1970.
  • Object-Oriented Model — data is kept as objects, like in OOP programming.
  • E-R Model — used for database design, shows entities and relationships between them.
याद रखें: आजकल लगभग सभी popular DBMS (MySQL, Oracle, SQL Server) Relational Model पर आधारित होते हैं। Remember: Nowadays almost all popular DBMS (MySQL, Oracle, SQL Server) are based on the Relational Model.

Schema और Instance

Schema and Instance

  • Schema — database की overall design/structure (जो बार-बार नहीं बदलती)
  • Instance — किसी particular समय पर database में मौजूद actual data (जो लगातार बदलता रहता है)
  • Schema — the overall design/structure of the database (which doesn't change often)
  • Instance — the actual data present in the database at a particular time (which keeps changing)

Data Independence

Data Independence

इसका मतलब है कि database की internal structure बदलने पर application programs पर असर न पड़े।

This means that changing the database's internal structure should not affect application programs.

  • Physical Data Independence — storage structure बदलने पर logical schema पर असर नहीं पड़ता
  • Logical Data Independence — logical schema बदलने पर application programs पर असर नहीं पड़ता
  • Physical Data Independence — changing storage structure does not affect the logical schema
  • Logical Data Independence — changing the logical schema does not affect application programs

Unit 03 · डिज़ाइनUnit 03 · Design

ER Model (Entity-Relationship Model)ER Model (Entity-Relationship Model)

ER Model क्या है?

What is ER Model?

ER Model database design के लिए इस्तेमाल होने वाला एक graphical तरीका है जिसमें real-world चीज़ों (entities) और उनके आपसी संबंधों को diagram के रूप में दिखाया जाता है। इसे ER Diagram (ERD) भी कहते हैं।

The ER Model is a graphical method used for database design, in which real-world things (entities) and the relationships between them are shown in a diagram. It is also called an ER Diagram (ERD).

Entity क्या है?

What is an Entity?

Entity एक real-world object या concept है जिसके बारे में database में data store किया जाता है। जैसे — Student, Teacher, Book, Car।

An Entity is a real-world object or concept about which data is stored in a database. For example — Student, Teacher, Book, Car.

  • Entity Set — एक जैसी properties वाली entities का group (जैसे सभी Students)
  • Weak Entity — जिस entity की अपनी primary key नहीं होती, वो किसी दूसरी entity पर निर्भर रहती है
  • Entity Set — a group of entities with similar properties (e.g., all Students)
  • Weak Entity — an entity that doesn't have its own primary key, and depends on another entity

Attribute क्या है?

What is an Attribute?

Attribute किसी entity की property या characteristic होती है। जैसे Student entity के attributes — Name, Roll Number, Age।

An Attribute is a property or characteristic of an entity. For example, the Student entity's attributes — Name, Roll Number, Age.

Relationship क्या है?

What is a Relationship?

Relationship बताता है कि दो या ज़्यादा entities आपस में कैसे जुड़ी हुई हैं। जैसे — "Student enrolls in Course"।

A Relationship describes how two or more entities are connected to each other. For example — "Student enrolls in Course".

Relationship के प्रकार (Cardinality)

Types of Relationship (Cardinality)

  • One-to-One (1:1) — एक entity का दूसरी entity से सिर्फ एक संबंध। जैसे एक व्यक्ति का एक ही Passport।
  • One-to-Many (1:M) — एक entity कई entities से जुड़ी हो सकती है। जैसे एक Teacher कई Students को पढ़ाता है।
  • Many-to-One (M:1) — कई entities एक entity से जुड़ी हों। जैसे कई Students एक Class में।
  • Many-to-Many (M:N) — कई entities कई entities से जुड़ी हों। जैसे Students कई Courses लेते हैं और एक Course में कई Students होते हैं।
  • One-to-One (1:1) — one entity has only one connection to another entity. Example: a person has only one Passport.
  • One-to-Many (1:M) — one entity can be connected to many entities. Example: one Teacher teaches many Students.
  • Many-to-One (M:1) — many entities connect to one entity. Example: many Students belong to one Class.
  • Many-to-Many (M:N) — many entities connect to many entities. Example: Students take many Courses and a Course has many Students.

Relational Model (रिलेशनल मॉडल)Relational Model

Relational Model क्या है?

What is Relational Model?

Relational Model में data को Tables (Relations) के रूप में रखा जाता है। हर table में rows और columns होते हैं। इसे E.F. Codd ने 1970 में propose किया था।

In the Relational Model, data is stored as Tables (Relations). Every table has rows and columns. It was proposed by E.F. Codd in 1970.

ज़रूरी Terminology

Important Terminology

TermमतलबMeaning

Relational Model की Properties

Properties of Relational Model

  • हर cell में सिर्फ एक ही value होती है (Atomic value)
  • हर column का एक unique नाम होता है
  • एक table में सभी rows unique होती हैं (कोई duplicate row नहीं)
  • rows और columns का कोई fixed order नहीं होता
  • Every cell has only one value (Atomic value)
  • Every column has a unique name
  • All rows in a table are unique (no duplicate rows)
  • Rows and columns have no fixed order

Integrity Constraints (अखंडता नियम)

Integrity Constraints

  • Entity Integrity — Primary Key कभी भी NULL नहीं हो सकती
  • Referential Integrity — Foreign Key का value या तो NULL होगा या किसी existing Primary Key से match होगा
  • Domain Integrity — Column की value उसके defined domain के अनुसार ही होनी चाहिए
  • Entity Integrity — Primary Key can never be NULL
  • Referential Integrity — Foreign Key's value must either be NULL or match an existing Primary Key
  • Domain Integrity — a column's value must match its defined domain

Keys in DBMS (कुंजियाँ)Keys in DBMS

Key क्या है?

What is a Key?

Key एक या एक से ज़्यादा attributes का set होता है जो किसी table की किसी row (tuple) को uniquely identify करने में मदद करता है।

A Key is a set of one or more attributes that help to uniquely identify a row (tuple) in a table.

Keys के प्रकार

Types of Keys

KeyपरिभाषाDefinition
Candidate Keyवो सभी attributes जो किसी row को uniquely identify कर सकते हैंAll attributes that can uniquely identify a row
Primary KeyCandidate Keys में से चुनी गई एक key जिसे row identify करने के लिए use किया जाता है — यह NULL नहीं हो सकतीOne key chosen from Candidate Keys, used to identify a row — cannot be NULL
Alternate Keyबाकी के candidate keys जो primary key नहीं बनींThe remaining candidate keys that didn't become the primary key
Foreign Keyएक table का attribute जो दूसरी table की primary key को refer करता हैAn attribute of one table that refers to another table's primary key
Composite Keyदो या ज़्यादा attributes मिलकर बनाई गई key जो row को uniquely identify करती हैA key made of two or more attributes that uniquely identifies a row

याद रखने का आसान तरीका: Super Key ⊃ Candidate Key → Primary Key चुनी जाती है, बाकी Alternate Key बन जाती हैं। Easy memory tip: Super Key ⊃ Candidate Key → one is chosen as Primary Key, the rest become Alternate Keys.

उदाहरण — Foreign Key

Example — Foreign Key

Student table में StudentID Primary Key है। Marks table में StudentID column है जो Student table के StudentID को refer करता है — यह Marks table में Foreign Key कहलाएगा।

In the Student table, StudentID is the Primary Key. In the Marks table, there's a StudentID column that refers to the Student table's StudentID — this is called a Foreign Key in the Marks table.

Normalization (नॉर्मलाइज़ेशन)Normalization

Normalization क्या है?

What is Normalization?

Normalization एक process है जिसमें database के tables को इस तरह design किया जाता है कि data redundancy कम हो और data integrity बनी रहे।

Normalization is a process in which database tables are designed so that data redundancy is reduced and data integrity is maintained.

Normalization क्यों ज़रूरी है?

Why is Normalization Needed?

  • Data की duplicate copies को कम करता है
  • Insert, Update, Delete में होने वाली anomalies को रोकता है
  • Database की storage space बचाता है
  • Data को consistent और accurate रखता है
  • Reduces duplicate copies of data
  • Prevents anomalies that occur during Insert, Update, Delete
  • Saves database storage space
  • Keeps data consistent and accurate

Anomalies जो बिना Normalization के होती हैं

Anomalies Without Normalization

Anomalyसमस्याProblem
Insertion Anomalyनया data डालने में दिक्कत आती हैDifficulty in inserting new data
Update Anomalyएक value को update करने पर कई जगह बदलाव करना पड़ता हैUpdating one value requires changes in multiple places
Deletion Anomalyएक record delete करने से कभी-कभी ज़रूरी data भी चला जाता हैDeleting a record sometimes removes necessary data too

1NF — First Normal Form

1NF — First Normal Form

नियम: हर column में atomic (अविभाज्य) value होनी चाहिए — कोई multi-valued या repeating group नहीं होना चाहिए।

Rule: every column must have an atomic value — no multi-valued or repeating groups allowed.

2NF — Second Normal Form

2NF — Second Normal Form

नियम: Table पहले 1NF में होनी चाहिए, और सभी non-key attributes पूरी Primary Key पर fully functionally dependent होने चाहिए (कोई Partial Dependency नहीं)।

Rule: the table must first be in 1NF, and all non-key attributes must be fully functionally dependent on the entire Primary Key (no Partial Dependency).

3NF — Third Normal Form

3NF — Third Normal Form

नियम: Table पहले 2NF में होनी चाहिए, और कोई Transitive Dependency नहीं होनी चाहिए।

Rule: the table must first be in 2NF, and there should be no Transitive Dependency.

BCNF — Boyce-Codd Normal Form

BCNF — Boyce-Codd Normal Form

3NF का strict version। नियम: हर Functional Dependency में, बायीं ओर वाला attribute (Determinant) एक Super Key होना चाहिए।

A stricter version of 3NF. Rule: in every Functional Dependency, the left-side attribute (Determinant) must be a Super Key.

Normal Formमुख्य शर्तMain Condition
1NFAtomic values, कोई repeating group नहींAtomic values, no repeating groups
2NF1NF + कोई Partial Dependency नहीं1NF + no Partial Dependency
3NF2NF + कोई Transitive Dependency नहीं2NF + no Transitive Dependency
BCNF3NF + हर Determinant एक Super Key हो3NF + every Determinant must be a Super Key

Functional Dependency क्या है?

What is Functional Dependency?

अगर attribute A की value से attribute B की value निश्चित होती है, तो कहते हैं B, A पर functionally dependent है — इसे A → B लिखा जाता है। जैसे: Roll_No → Name।

If the value of attribute A determines the value of attribute B, then B is said to be functionally dependent on A — written as A → B. Example: Roll_No → Name.

Joins & FunctionsJoins & Functions

SQL Joins क्या हैं?

What are SQL Joins?

Join का इस्तेमाल दो या ज़्यादा tables का data एक साथ जोड़कर निकालने के लिए होता है, आमतौर पर एक common column के आधार पर।

A Join is used to combine data from two or more tables, usually based on a common column.

Join Typeक्या करता हैWhat it does
INNER JOINसिर्फ वो rows देता है जो दोनों tables में match करती हैंReturns only rows that match in both tables
LEFT JOINबायीं table की सारी rows + matching rows दायीं table सेAll rows from left table + matching rows from right table
RIGHT JOINदायीं table की सारी rows + matching rows बायीं table सेAll rows from right table + matching rows from left table
FULL JOINदोनों tables की सारी rows, चाहे match हों या नहींAll rows from both tables, matched or not
SELF JOINएक table को खुद के साथ join करनाJoining a table with itself
-- INNER JOIN उदाहरण-- INNER JOIN example SELECT Student.Name, Marks.Marks FROM Student INNER JOIN Marks ON Student.Roll_No = Marks.Roll_No;

GROUP BY और HAVING

GROUP BY and HAVING

GROUP BY rows को groups में बांटता है। HAVING उन groups पर condition लगाता है (WHERE की तरह, पर groups के लिए)।

GROUP BY divides rows into groups. HAVING applies a condition on those groups (like WHERE, but for groups).

SELECT Class, AVG(Marks) FROM Student GROUP BY Class HAVING AVG(Marks) > 75;

Subquery (उप-प्रश्न)

Subquery

एक query के अंदर दूसरी query — पहले अंदर वाली query चलती है, फिर उसका result बाहर वाली query इस्तेमाल करती है।

A query inside another query — the inner query runs first, then its result is used by the outer query.

SELECT Name FROM Student WHERE Marks > (SELECT AVG(Marks) FROM Student);

Transactions & ACID PropertiesTransactions & ACID Properties

Transaction क्या है?

What is a Transaction?

Transaction database पर किए गए operations का एक logical unit होता है, जो या तो पूरी तरह complete होता है या बिल्कुल नहीं होता। जैसे बैंक में पैसे transfer करना।

A Transaction is a logical unit of operations performed on a database, which either completes fully or not at all. Example: transferring money in a bank.

Transaction के States

States of a Transaction

  • Active — Transaction चल रहा है
  • Partially Committed — आखिरी operation पूरा हो गया है
  • Committed — Transaction सफलतापूर्वक पूरा हो गया
  • Failed — Transaction में कोई error आ गया
  • Aborted — Transaction को rollback कर दिया गया
  • Active — the transaction is running
  • Partially Committed — the last operation has completed
  • Committed — the transaction has completed successfully
  • Failed — an error occurred in the transaction
  • Aborted — the transaction has been rolled back

ACID Properties

ACID Properties

PropertyमतलबMeaning
AtomicityTransaction या तो पूरा होगा या बिल्कुल नहीं होगाTransaction either completes fully or not at all
ConsistencyTransaction के पहले और बाद database हमेशा valid state में रहेगाDatabase remains in a valid state before and after the transaction
Isolationएक साथ चल रहे कई transactions एक-दूसरे को प्रभावित नहीं करेंगेMultiple transactions running simultaneously won't affect each other
Durabilityएक बार commit होने के बाद changes permanent रहेंगेOnce committed, changes remain permanent

Concurrency ControlConcurrency Control

Concurrency Control क्यों ज़रूरी है?

Why is Concurrency Control Needed?

जब कई users एक साथ database को access करते हैं तो data में गड़बड़ी हो सकती है। Concurrency Control techniques इसे रोकती हैं।

When multiple users access a database simultaneously, data inconsistencies can occur. Concurrency Control techniques prevent this.

Concurrency से होने वाली समस्याएं

Problems Caused by Concurrency

समस्याProblemक्या होता हैWhat happens
Lost Updateदो transactions एक साथ data update करते हैं और एक का update खो जाता हैTwo transactions update data simultaneously and one's update is lost
Dirty Readएक transaction दूसरे का uncommitted data पढ़ लेता हैA transaction reads another's uncommitted data
Unrepeatable Readएक transaction में बार-बार पढ़ने पर अलग-अलग value मिलती हैReading repeatedly within one transaction gives different values

Concurrency Control की Techniques

Concurrency Control Techniques

  • Lock-Based Protocol — data पर lock लगाकर एक समय में सिर्फ एक transaction को access देना
  • Time-Stamping Protocol — हर transaction को एक unique timestamp देकर उनका order तय करना
  • Optimistic Control — मान लेना कि conflict कम होगा, और सिर्फ अंत में check करना
  • Lock-Based Protocol — gives access to only one transaction at a time by locking the data
  • Time-Stamping Protocol — gives every transaction a unique timestamp to fix their order
  • Optimistic Control — assumes conflicts will be rare, checks only at the end

Deadlock क्या है?

What is a Deadlock?

Deadlock तब होता है जब दो या ज़्यादा transactions एक-दूसरे के resource को पाने के लिए wait करते रहते हैं और कोई भी आगे नहीं बढ़ पाता।

A Deadlock occurs when two or more transactions keep waiting for each other's resources, and none can proceed.

Indexing क्या है?

What is Indexing?

Index एक data structure है जो database में data को तेज़ी से ढूंढने में मदद करता है — ठीक उसी तरह जैसे किताब के Index से कोई topic जल्दी मिल जाता है।

An Index is a data structure that helps search data faster in a database — just like a book's Index helps find a topic quickly.

File Organization Methods

  • Sequential File Organization — records को क्रम में, एक के बाद एक store करना

  • Sequential File Organization — stores records in order, one after another
  • Heap File Organization — stores records anywhere in available free space, with no order
  • Hash File Organization — finds a record's location using a hash function

Backup, Recovery & SecurityBackup, Recovery & Security

Backup क्या है?

What is Backup?

Backup database की एक copy बनाने की process है ताकि अगर original data खराब हो जाए या खो जाए, तो उसे वापस पाया जा सके।

Backup is the process of making a copy of the database so that if the original data is damaged or lost, it can be recovered.

Backup के प्रकार

Types of Backup

  • Full Backup — पूरे database की पूरी copy
  • Incremental Backup — सिर्फ वो data जो पिछले backup के बाद बदला है
  • Differential Backup — आखिरी full backup के बाद का सारा बदला हुआ data
  • Full Backup — a complete copy of the entire database
  • Incremental Backup — only the data that changed since the last backup
  • Differential Backup — all changed data since the last full backup

Recovery क्या है?

What is Recovery?

System crash या failure होने के बाद database को वापस एक consistent state में लाने की process को Recovery कहते हैं। इसमें Log files का इस्तेमाल होता है।

Recovery is the process of bringing the database back to a consistent state after a system crash or failure. Log files are used for this.

Database Security

Database Security

  • Authentication — username/password से user की पहचान verify करना
  • Authorization — user को सिर्फ उतनी permission देना जितनी ज़रूरी है
  • Encryption — data को code में बदलना ताकि कोई बिना key के पढ़ न सके
  • Views — user को सिर्फ ज़रूरी data दिखाना, पूरा table नहीं
  • Authentication — verifying user identity via username/password
  • Authorization — giving users only as much permission as necessary
  • Encryption — converting data into code so no one can read it without the key
  • Views — showing users only necessary data, not the whole table
